SuperMarker3D

Inherits: Node3D < Node < Object

Graphic-design-oriented 3D marker node — five marker types (Axis, Mesh, Shape, Curve, Figure) covering 26 subtype variants, with universal outline / fill / detail-mode controls.

Description

SuperMarker3D is a standalone 3D marker / annotation node. Drop one into any scene as a Node3D child; everything is configured from the inspector or script. No companion plugin required — though it cooperates with MultiNode as a template child (set template_mode via set_template_mode and the instancer will draw per-instance copies).

Type / subtype model

The top-level type dropdown picks a MarkerType category. The subtype dropdown then narrows to the variants in that category — CROSS / PLAIN / BURR / XYZ for Axis, SPHERE / BOX / DIAMOND / CYLINDER / CONE / CAPSULE for Mesh, CIRCLE / SQUARE / DIAMOND / TRIANGLE / CAPSULE / X / ARROW_FLAT for Shape, LINE / RIGHT_ANGLE / ARC / SINE / HELIX / BEZIER / CUSTOM for Curve, and FIGURE for Figure. The inspector hides properties that don't apply to the current type.

Detail modes

DETAIL_WIREFRAME — filled solid mesh (back-culled) with thin face-normal edge quads, so only camera-front-facing edges are visible. Looks like a real mesh at any angle.

DETAIL_SILHOUETTE — billboarded 2D outline + optional fill, always faces the camera.

API stability

Pre-1.0: enum reordering, name changes, and removals are permitted between minor versions. From 1.0 forward: enum integer values are frozen for scene-file compatibility, no renames, no removals without a deprecation cycle.

Minimal script example

var m := SuperMarker3D.new()
m.type = SuperMarker3D.TYPE_MESH
m.subtype = SuperMarker3D.MESH_SPHERE
m.marker_size = 0.75
m.outline_color = Color.YELLOW
m.fill_color = Color(0.0, 0.6, 1.0, 0.5)
add_child(m)

Enumerations

enum MarkerType:

  • TYPE_AXIS = 0 — Axis category — line clusters through the origin (Cross / Plain / Burr / XYZ).
  • TYPE_MESH = 1 — Mesh category — closed 3D primitives.
  • TYPE_SHAPE = 2 — Shape category — flat 2D polygon icons + the flat arrow.
  • TYPE_CURVE = 3 — Curve category — path stamped along a Curve3D.
  • TYPE_FIGURE = 5 — Figure category — humanoid mock-up with an 11-bone rig.

enum Subtype:

Integer values are frozen for scene-file compatibility — listed in numeric order below.

  • AXIS_CROSS = 0 — Four arms on the X/Y plane (Z disabled).
  • MESH_DIAMOND = 1 — Octahedron.
  • MESH_SPHERE = 2 — UV sphere.
  • AXIS_PLAIN = 3 — Six axes, all rendered in outline_color.
  • MESH_BOX = 4 — Axis-aligned cube.
  • ARROW_FLAT = 6 — Flat 2D arrow icon.
  • AXIS_XYZ = 8 — Six axes with per-direction colors.
  • FIGURE = 10 — Humanoid figure.
  • AXIS_BURR = 11 — Six cardinal + six face-diagonal axes (12 lines).
  • MESH_CYLINDER = 14 — Capped cylinder.
  • MESH_CONE = 15 — Round-base cone (mesh_sides = 4 → pyramid).
  • MESH_CAPSULE = 16 — Cylinder body + hemisphere caps.
  • FLAT_CIRCLE = 17 — Regular N-gon.
  • FLAT_SQUARE = 18 — Axis-aligned square.
  • FLAT_DIAMOND = 19 — Rhombus.
  • FLAT_TRIANGLE = 20 — Equilateral triangle, apex up.
  • FLAT_CAPSULE = 21 — 2D pill.
  • FLAT_X = 22 — X / close icon.
  • CURVE_LINE = 23 — Straight segment.
  • CURVE_RIGHT_ANGLE = 24 — L-bend.
  • CURVE_ARC = 25 — Circular arc.
  • CURVE_SINE = 26 — Sine wave.
  • CURVE_HELIX = 27 — 3D coil.
  • CURVE_BEZIER = 28 — Smooth S-curve.
  • CURVE_CUSTOM = 29 — User-supplied Curve3D.

enum AxisLinkMode:

  • LINK_ALL = 0 — A single value drives all six axis lengths.
  • LINK_MIRRORED = 1 — Pos and neg of the same axis share a value.
  • LINK_FREE = 2 — Every direction is independent.

enum DetailMode:

  • DETAIL_WIREFRAME = 0 — Filled mesh + face-normal edge quads.
  • DETAIL_SILHOUETTE = 1 — Billboarded 2D outline + optional fill.

enum CurvePattern:

  • CURVE_PATTERN_SOLID = 0
  • CURVE_PATTERN_DASH = 1
  • CURVE_PATTERN_DOT = 2

enum CurveCapStyle:

  • CURVE_CAP_NONE = 0
  • CURVE_CAP_ARROW = 1
  • CURVE_CAP_DOT = 2
  • CURVE_CAP_LINE = 3

enum FigureBone:

  • BONE_PELVIS = 0 — Root bone; position from figure_bone_pelvis_pos.
  • BONE_SPINE = 1
  • BONE_HEAD = 2
  • BONE_L_UPPER_ARM = 3
  • BONE_L_LOWER_ARM = 4
  • BONE_R_UPPER_ARM = 5
  • BONE_R_LOWER_ARM = 6
  • BONE_L_UPPER_LEG = 7
  • BONE_L_LOWER_LEG = 8
  • BONE_R_UPPER_LEG = 9
  • BONE_R_LOWER_LEG = 10
  • BONE_COUNT = 11 — Sentinel.

Property Descriptions

MarkerType type#

void set_type ( int value )
int get_type ( )

Top-level marker category. Picking a type narrows the subtype dropdown to variants in that category, and toggles the inspector visibility of every type-specific group below. Changing type re-initialises type-specific data fields to their defaults.

Subtype subtype#

void set_subtype ( int value )
int get_subtype ( )

Variant within the current type. Enum integer values are frozen for scene-file compatibility — old pre-1.0 scenes store these directly and keep their meaning.

Color outline_color#

Universal edge / outline color. Used by every subtype that draws an outline strip.

float outline_thickness#

Universal outline width in meters. At 0.0, outlines render as 1px lines (crisp at every camera distance). Above 0, axis arms become thin tubes, arrowheads gain volume, and mesh-face outlines paint as a strip of the given thickness.

float marker_size#

Overall size in meters for Mesh and Shape subtypes. Hidden for Axis (uses per-direction lengths) and Figure (uses figure_height).

float capsule_height#

Body length between the two hemisphere caps for MESH_CAPSULE, and the body length of the 2D pill for FLAT_CAPSULE. Cap radius follows marker_size.

DetailMode detail_mode#

DETAIL_WIREFRAME for a real-mesh look with face-normal edges; DETAIL_SILHOUETTE for a billboarded 2D outline that always faces the camera.

Color fill_color#

Solid-body color. Alpha < 1 makes the marker translucent; alpha = 0 hides the fill entirely (outline-only).

Color background_color#

Curve subtypes only: color of the gap panels between dashes / dots. Transparent by default so gaps are invisible.

AxisLinkMode axis_link_mode#

Linkage between the six axis_length_* fields. LINK_ALL drives all six from a single value (uniform marker). LINK_MIRRORED couples each axis's pos and neg directions. LINK_FREE leaves every direction independent. Shared across every Axis subtype so switching between Cross / Plain / Burr / XYZ preserves authored values and link behavior.

float axis_length_x_pos#

Length of the +X axis arm. Set to 0 to hide this arm. See axis_link_mode for which fields are slaved.

float axis_length_x_neg#

Length of the -X axis arm. Set to 0 to hide this arm.

float axis_length_y_pos#

Length of the +Y axis arm. Set to 0 to hide this arm.

float axis_length_y_neg#

Length of the -Y axis arm. Set to 0 to hide this arm.

float axis_length_z_pos#

Length of the +Z axis arm. Set to 0 to hide this arm.

float axis_length_z_neg#

Length of the -Z axis arm. Set to 0 to hide this arm. Note: AXIS_CROSS disables Z entirely.

bool axis_arrows#

Add a cone arrowhead to the tip of every non-zero axis arm. Honored by AXIS_CROSS, AXIS_PLAIN, and AXIS_XYZ; ignored by AXIS_BURR (the diagonals make it visually noisy).

float axis_arrow_length#

Arrowhead cone height in meters, measured back from the arm tip along the arm direction.

float axis_arrow_width#

Arrowhead base radius (splay perpendicular to the arm).

Color axis_color_x_pos#

AXIS_XYZ only: bright red on the +X arm by default. Per-arm color survives the line→tube switch at thickness > 0.

Color axis_color_x_neg#

AXIS_XYZ only: dark red on the -X arm by default.

Color axis_color_y_pos#

AXIS_XYZ only: bright green on the +Y arm by default.

Color axis_color_y_neg#

AXIS_XYZ only: dark green on the -Y arm by default.

Color axis_color_z_pos#

AXIS_XYZ only: bright blue on the +Z arm by default.

Color axis_color_z_neg#

AXIS_XYZ only: dark blue on the -Z arm by default.

float figure_height#

Standing tip-to-toe extent of the Figure subtype, in meters. Body proportions are derived from this value.

bool figure_show_mesh#

When true, the skinned humanoid mesh renders. Turn off to see only the rig bones (combine with figure_show_bones).

bool figure_show_bones#

Rigging mode toggle. When true: pose rotations are forced to zero, per-bone rest rotation / length / width and the offsets become inspector-visible, and a tube overlay of every bone renders on top of the rest mesh. When false: pose rotations drive the chain, the mesh renders posed, and rig fields are hidden.

Color figure_bone_color#

Color of the bone-overlay tubes drawn when figure_show_bones is on. Independent from fill / outline color. Alpha < 1 lets you see bones through the mesh while rigging.

Vector3 figure_bone_pelvis_pos#

Pelvis position in the marker's local space (rigging-only, baked). All other bones cascade from this point through the rig graph.

Note: Each of the 10 non-pelvis bones exposes a parallel _rot, _length, _width, and _pose_rot property (e.g. figure_bone_l_upper_arm_rot). The first three are rigging-only (locked once the rig is baked); _pose_rot is the runtime animation control — zero means rest pose, anything else rotates the bone (and every descendant) on top of the rest.
Note: The five offset bars — figure_offset_head_base, figure_offset_l_shoulder, figure_offset_r_shoulder, figure_offset_l_hip, figure_offset_r_hip — attach child pivots to their parent bone's tip in the parent's rotated frame. Each has a paired _width for the capsule radius used by the skinning Voronoi.

float head_length#

ARROW_FLAT only: arrowhead length in marker-local units, measured back from the arrow tip.

float head_width#

ARROW_FLAT only: full arrowhead width at the shoulders.

bool curve_flat#

Curve only: when true the curve renders as a flat ribbon (2D-look stamp on a 3D path); when false it renders as a 3D tube.

float curve_length#

Curve preset only: overall length of the generated path (semantics vary slightly by preset — straight-line distance for CURVE_LINE, arc chord, helix axial length, etc.).

float curve_amplitude#

Curve preset only: bend / radius / wave depth for presets that have one (arc radius, sine height, helix radius, bezier deflection).

float curve_turns#

Curve preset only: cycle count for periodic presets (sine cycles, helix turns).

int curve_segments#

Curve preset only: tessellation segment count along the path. Higher = smoother but more verts.

Curve3D curve#

User-supplied Curve3D driving the path when subtype is CURVE_CUSTOM. Ignored for the procedural presets — see get_active_curve to get the actual curve in use.

float curve_width#

Curve only: ribbon / tube width (flat) or tube radius (3D) in meters.

float curve_bank#

Curve only: lean angle of the ribbon around its travel direction, in radians. Useful for banking a racetrack ribbon into turns.

float bank_easing#

Curve only: ease-in / ease-out fraction at the ends of the bank ramp (0 = no easing, hard step; 0.5 = full sigmoid).

CurvePattern curve_pattern#

Curve only: CURVE_PATTERN_SOLID, CURVE_PATTERN_DASH, or CURVE_PATTERN_DOT.

float dash_length#

Curve only: length of one dash segment when curve_pattern is DASH.

float dash_gap#

Curve only: length of the gap between dashes / dots.

CurveCapStyle curve_start_cap#

Curve only: cap style at the start of the path — None / Arrow / Dot / Line.

Vector2 start_cap_size#

Curve only: cap dimensions at the start. x = along-path length, y = perpendicular width. When start_cap_linked is true, the setter mirrors x into y so the cap stays proportional.

bool start_cap_linked#

Curve only: when true, start_cap_size keeps x and y synchronized.

CurveCapStyle curve_end_cap#

Curve only: cap style at the end of the path.

Vector2 end_cap_size#

Curve only: cap dimensions at the end. See start_cap_size.

bool end_cap_linked#

Curve only: when true, end_cap_size keeps x and y synchronized.

float length_fraction#

Curve only: fraction (0–1) of the path that is actually rendered. Useful for "drawing in" a path over time — animate from 0 to 1 with a Tween.

int mesh_sides#

Mesh only: longitudinal segment count for sphere / diamond / cone / cylinder / capsule. Range 5..24. Controls both fill tessellation and wireframe segmentation. Use mesh_sides = 4 on MESH_CONE for a four-sided pyramid.

bool smooth_shading#

Mesh only: when true (default), curved subtypes use smooth per-pixel normals plus the analytical sphere shader for lat/lon (or meridian + rim) outlines. When false, every face uses its flat face normal and outlines are painted along the actual mesh edges — the faceted, low-poly look. Pyramid is always faceted regardless.

bool two_sided#

Render both sides of the geometry. Back faces draw in an earlier pass (cull_front, no depth-write, lower render_priority) and front faces overwrite them where they overlap — so the mesh reads correctly from all angles at any fill alpha. Flat-shape subtypes use a single cull-disabled surface instead.

bool flip_faces#

When two_sided is false, renders the back face instead of the front face.

bool checker_enabled#

Checkerboard overlay. When on, multiplies a darken factor into alternate squares of a triplanar grid in marker-local space — useful for showing real-world scale on Mesh / Shape / Curve / Figure / Axis tubes while prototyping a level.

float checker_size#

Checker only: edge length of one square, in the same units as the marker's local space.

float checker_darken#

Checker only: multiplier applied to alternating squares (0 = black, 1 = no change).

bool billboard_xz#

Shape only: when true, the flat shape rotates in the XZ plane to face the camera while keeping Y axis fixed (BILLBOARD_FIXED_Y). Mutually distinct from billboard_y.

bool billboard_y#

Shape only: when true, the flat shape fully faces the camera (BILLBOARD_ENABLED).

bool rounded_corners#

Shape only: when true, arc joints and polygon corners gain a disc blob to produce smooth rounded joins.

int shape_sides#

FLAT_CIRCLE only: regular N-gon segment count. Range 6..64. Hidden for all other subtypes.

bool shows_in_play#

Renderer flag — when false, the marker hides at runtime (only visible in the editor). Pair with always_on_top for editor-only annotations.

bool always_on_top#

Renderer flag — when true, disables depth testing so the marker draws over opaque geometry. Useful for editor cues and HUD overlays.

bool lights_and_shadows#

Renderer flag — when true, the marker switches to per-pixel lit shading and casts / receives shadows like a normal MeshInstance3D. A thick burr stands in for a bush, a tall mesh box stands in for a building, etc. Off (default) = unshaded, no shadows — best for editor cues, HUD overlays, and pure design markers.

Method Descriptions

ArrayMesh export_mesh ( ) const#

Returns a freestanding ArrayMesh duplicate of every surface currently rendered by this marker (the primary mesh and, for Axis subtypes, each per-arm surface). Drop the result into a MeshInstance3D, save it to disk, etc. Geometry is in marker-local space — apply the marker's transform when consuming the result if you want world-space. No materials are attached.

var marker := $SuperMarker3D
var mesh := marker.export_mesh()
$MeshInstance3D.mesh = mesh
ResourceSaver.save(mesh, "res://my_marker.tres")

ConvexPolygonShape3D export_convex_shape ( ) const#

Convex-hull collision shape covering every triangle vertex of the marker's geometry. Cheap and stable at runtime; concavities are filled in. Suitable for movable physics bodies (RigidBody3D / CharacterBody3D).

ConcavePolygonShape3D export_concave_shape ( ) const#

Triangle-mesh collision shape that matches the marker's visuals exactly — keeps helix coils, right-angle bends, arc curves, etc. Static-only (no movement under physics) and pricier per query than the convex variant.

Curve3D get_active_curve ( ) const#

Returns a duplicate of the Curve3D currently driving the geometry — the user-supplied curve resource for CURVE_CUSTOM, or a freshly generated curve built from the active preset's parameters for any other Curve subtype. The duplicate is independent: edit it, save it as a .tres, or hand it to a Path3D for an AI agent / camera to follow without affecting the marker. Returns an empty Curve3D if the marker isn't a Curve subtype.

var path := Path3D.new()
path.curve = $SuperMarker3D.get_active_curve()
add_child(path)
var follower := PathFollow3D.new()
path.add_child(follower)
# Now `follower` traces the marker's curve.

RID get_mesh_rid ( ) const#

Returns the RenderingServer mesh RID currently held by this marker. Useful when wiring a SuperMarker3D into another instancer (e.g. MultiNode's per-instance mesh ID slot).

bool is_template_mode ( ) const#

Returns true when this marker has been flipped into template mode by an external instancer — see set_template_mode.

void reset_interpolation ( )#

Reset the physics interpolation state so the next render frame draws at the current transform with no blend from the previous one. Call after teleporting this marker (or its parent) to a new pose — otherwise the engine will draw a one-frame smear from the old position to the new one.

void set_template_mode ( bool template_mode )#

Flip the marker into "template" mode — it stops rendering itself, but its mesh / materials remain available for an external instancer to draw. MultiNode's SuperMarkerHandler calls this at scan time so a SuperMarker3D used as a template child doesn't double-render alongside its stamped instances. Not normally called from game code.
